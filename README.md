## 스토리

**가문의 전통을 깨부수는, 작고 어리숙한 '걸작'을 만드세요!**

<p align="center">
  <img src="./assets/GoGoGolem_Home.gif" alt="게임 주요 장면 1" width="800"/>
</p>

대대로 마을을 수호하는 **완벽한 골렘**을 만들어야 하는 장인 가문의 후계자, 그리고 그 손에서 태어난 **예상 밖의 골렘**이 함께 성장하는 판타지 어드벤처입니다.

제작 중 실수로 탄생한 이 **작고 호기심 많은 친구**는 주인공과 **영혼이 연결**되어 버립니다. 사람과 골렘의 영혼 연결은 마을의 금기. 당신은 이 모자란 듯 특별한 골렘과 함께 여정을 떠나 **음성, 손짓, 편지**로 교감하며, 낯선 정착지의 주민들을 돕고 진정한 수호자로 거듭납니다.

**과연 당신의 '실수'는 마을의 영웅이 될 수 있을까요?**

### 당신의 목소리와 몸짓으로 완성되는, 새로운 형태의 어드벤처입니다.

마이크와 웹캠을 통해, 골렘과 직접 대화하고 교감하세요.
말을 걸고, 손을 뻗고, 신호를 보내며 — 컨트롤러 너머의 행동이 그대로 게임 속 세계에 이어집니다.

**작고 어리숙한 골렘과 함께, 정착지의 삶 속으로 들어가세요**

말을 잘 못하는 골렘, 하지만 언제나 당신을 따라오는 존재. 완벽하지 않기에 더 특별한 이 친구와 함께, 하루하루를 만들어갑니다.

**마을 주민들의 의뢰를 통해 세상과 연결되세요.**

길 잃은 물건을 찾아달라는 할아버지, 메말라있는 마을 우물, 작은 부탁에서 시작된 여정은 점점 더 깊어지고, 당신은 이곳의 일부가 되어갑니다.


**당신만의 방식으로 골렘과 호흡을 맞추세요.**

같은 상황에서도, 어떻게 말하고 어떻게 움직이느냐에 따라 골렘의 반응과 행동은 달라집니다. 정답은 없습니다. 당신과 골렘이 만들어가는 방식이 곧 플레이 스타일이 됩니다.

**몸짓으로 완성되는 협동 마법을 익히세요.**

스킬을 발동시키기 위해 손동작을 하세요. 골렘은 당신의 움직임을 따라하며 힘을 끌어냅니다. 처음엔 서툴지만, 점점 더 자연스럽고 강력한 호흡으로 이어집니다.

**편지를 통해, 또 다른 이야기를 이어가세요.**

하루가 끝나면 부모님께 편지를 씁니다. 오늘 겪은 일, 골렘과의 순간들, 그리고 아직 풀리지 않은 질문들. 다음 날 도착하는 답장은 당신이 보지 못했던 시선과 새로운 힌트를 전해줍니다.

**당신의 선택이 정착지를 변화시킵니다.**

의뢰를 하나씩 해결할 때마다, 비어 있던 공간에 새로운 변화가 생깁니다. 사람들이 모이고, 이야기가 쌓이고, 낯설었던 곳이 점점 당신의 마을이 되어갑니다.

**따뜻하면서도 어딘가 세계를 탐험하세요.**

조용한 숲길, 낯선 정착지 — 모든 공간에는 아직 밝혀지지 않은 이야기들이 남아 있습니다. 당신과 골렘은 그 조각들을 하나씩 이어가게 됩니다.

**완벽하지 않은 존재가, 진짜 수호자가 되어가는 이야기.**

처음부터 완벽할 필요는 없습니다. 실수로 태어난 골렘, 그리고 아직 미숙한 당신. 그 둘이 함께 만들어가는 여정이 이 세계를 지키는 힘이 됩니다.

## 기술 스택

| Layer       | Stack |
|-------------|-------|
| **Client**  | Unity 6000.0.62f1 · C# · New Input System · Animation Rigging · Yarn Spinner · MediaPipe · WebCamTexture |
| **Server**  | Python 3.11 · FastAPI · WebSocket · LiteLLM (ModelRouter 기반 Fallback) |
| **AI**      | OpenAI Realtime API · LLM 편지 응답 생성 |
| **DB**      | Firebase Firestore (편지) · Unity PlayerPrefs / JSON (로컬 세이브) |
| **Infra**   | Docker · docker-compose |

### 아키텍처 특징

* **Unity**
  `Managers` 싱글톤과 **ScriptableObject 기반 이벤트 버스**를 활용해 시스템 간 결합도를 낮췄습니다.
  또한 **MVP(Presenter–View) 패턴**을 적용해 UI 로직을 명확히 분리했습니다.

* **AI 서버**
  **Clean Architecture + DDD**를 기반으로 Server / Application / Domain / Infra의 4계층 구조로 설계했습니다.
  Port 인터페이스를 통해 도메인 레이어가 특정 구현 기술에 의존하지 않도록 구성했습니다.

* **AIBridge**
  HTTP(`AIHttpClient`)와 WebSocket(`RealtimeWebSocketClient`)을 분리하여
  통신 방식에 따라 독립적인 클라이언트 구조를 설계했습니다.

## How to Install

### 필수 환경

| 항목 | 버전 |
|------|------|
| OS | Windows 10/11 |
| Unity | 6000.0.62f1 |
| Python | 3.13 이상 |
| [uv](https://github.com/astral-sh/uv) | 최신 버전 |
| Docker & Docker Compose | 최신 버전 |
| 마이크 · 웹캠 | 필수 장치 |

### 1. 저장소 클론

```bash
git clone https://github.com/ob1hnk/GoGoGolem.git
cd GoGoGolem
```

### 2. AI 서버 의존성 설치

```bash
cd GoGoGolem/src/ai

# uv가 없는 경우 먼저 설치
pip install uv

# 의존성 설치 (Python 3.13 가상환경 자동 생성)
uv sync
```

### 3. 환경 변수 설정

`GoGoGolem/src/ai/.env` 파일을 생성하고 아래 키를 입력합니다.

```env
OPENAI_API_KEY=sk-...
AWS_ACCESS_KEY_ID=...           # AWS Bedrock 사용 시 (선택)
AWS_SECRET_ACCESS_KEY=...       # AWS Bedrock 사용 시 (선택)
AWS_REGION_NAME=us-east-1       # AWS Bedrock 사용 시 (선택)
FIREBASE_CREDENTIALS_PATH=./gogo-golem-firebase-adminsdk-fbsvc-xxxxxx.json
```

Firebase Admin SDK 서비스 계정 키 파일(`.json`)은 Firebase 콘솔 → 프로젝트 설정 → 서비스 계정에서 발급받아 `GoGoGolem/src/ai/` 경로에 위치시킵니다.

### 4. MediaPipe Unity Plugin 설치

MediaPipe Unity Plugin은 용량 제한으로 인해 저장소에 모델 파일이 포함되어 있지 않습니다. 아래 절차에 따라 직접 다운로드해야 합니다.

1. Unity Editor에서 프로젝트(`GoGoGolem/src/unity/GoGoGolem`)를 엽니다.
2. 메뉴 **Window → MediaPipe → Download Model** 또는 MediaPipeUnity 패키지 설치 안내에 따라 필요한 모델(Hand Landmarker, Pose Landmarker)을 다운로드합니다.
3. 다운로드된 모델 파일(`.task`)이 `Assets/StreamingAssets/mediapipe/` 경로에 위치하는지 확인합니다.
4. 씬에서 `GestureSceneConfig` ScriptableObject를 열고 **Model Type**(Lite / Full / Heavy) 및 **Num Hands** 값을 확인합니다.

### 5. Unity AI 서버 주소 설정

Unity 프로젝트에서 AI 서버 엔드포인트를 지정합니다.

1. Unity Editor에서 `Assets/Scripts/Runtime/Multimodal/Config/` 경로의 `AIBridgeConfig` ScriptableObject를 선택합니다.
2. **Server URL** 필드에 AI 서버 주소를 입력합니다.
   - 로컬 실행 시: `ws://localhost:8000`
   - 원격 서버 사용 시: `wss://<서버 IP 또는 도메인>:8000`

## How to Build

### AI 서버 빌드 및 실행

#### 방법 A — 직접 실행 (개발용)

```bash
cd GoGoGolem/src/ai

# 서버 실행 (포트 8000)
make run
# 또는
uv run uvicorn interaction.server.app:app --host 0.0.0.0 --port 8000 --reload
```

#### 방법 B — Docker 컨테이너 실행 (운영용)

```bash
cd GoGoGolem/src/ai

# 이미지 빌드 및 컨테이너 실행
docker compose up --build -d

# 로그 확인
docker compose logs -f

# 서버 상태 확인
curl http://localhost:8000/health
```

서버가 정상적으로 기동되면 `http://localhost:8000/docs`에서 FastAPI 자동 생성 API 문서를 확인할 수 있습니다.

### Unity 클라이언트 빌드

1. Unity Hub에서 Unity **6000.0.62f1** 버전으로 `GoGoGolem/src/unity/GoGoGolem` 프로젝트를 엽니다.
2. **File → Build Settings**를 열고 플랫폼을 **Windows, Mac, Linux**로 설정합니다.
3. **Player Settings → Other Settings**에서 아래를 확인합니다.
   - Scripting Backend: **IL2CPP** (권장) 또는 Mono
   - Api Compatibility Level: **.NET Standard 2.1**
4. `Assets/Scenes/` 에서 빌드에 포함할 씬 목록을 확인합니다:
   - `StartScene` → `PrologueScene` → `ForestScene` → `SettlementScene` → `RoomScene`
5. **Build** 버튼을 눌러 빌드를 실행합니다.

> **주의**: MediaPipe 모델 파일이 `StreamingAssets`에 없으면 손동작 인식이 비활성화됩니다. 빌드 전 [MediaPipe 설치](#4-mediapipe-unity-plugin-설치) 절차를 완료하세요.

## How to Test

### AI 서버 단위 테스트

```bash
cd GoGoGolem/src/ai

# 코드 스타일 검사
make lint

# 코드 자동 수정
make fix
```

### 음성 / 편지 기능 통합 테스트

```bash
cd GoGoGolem/src/ai

# 음성 스트리밍 WebSocket 테스트
uv run python interaction/server/tests/test_speech_v1.py

# Realtime API 테스트
uv run python interaction/server/tests/test_realtime_v1.py
```

### 편지 힌트 위장도 평가 (LLM-as-judge)

편지 시스템에서 다음 퀘스트 힌트가 부모의 추억담으로 얼마나 자연스럽게 위장되는지를 1~5점으로 정량 평가합니다.

```bash
cd GoGoGolem/src/ai

# 평가 실행 (OPENAI_API_KEY 필요)
.venv/bin/python evaluation/hint_disguise_eval.py
```

결과 파일:

| 파일 | 설명 |
|------|------|
| `evaluation/hint_disguise_scores.png` | 케이스별 점수 bar chart |
| `evaluation/hint_disguise_results.json` | 생성 편지 전문 + 점수 + 채점 근거 |

평가 방법론 및 루브릭 상세 내용: [`GoGoGolem/src/ai/evaluation/README.md`](GoGoGolem/src/ai/evaluation/README.md)

## 데이터 및 데이터베이스

### Firebase Firestore

편지 데이터(부모 NPC 답장)를 클라우드에 저장·관리합니다.

- **컬렉션**: `letters` — 플레이어가 쓴 편지 및 NPC 답장 저장
- 서비스 계정 키(`gogo-golem-firebase-adminsdk-fbsvc-*.json`)가 서버 측에서 필요합니다.
- Firestore 프로젝트 연결 설정은 Unity 측 `Assets/google-services.json`에서 관리됩니다.

### 로컬 세이브

- 퀘스트 진행 상태, 인벤토리, 방 상태 등은 **Unity PlayerPrefs** 및 **JSON 파일** 형태로 클라이언트 로컬에 저장됩니다.

### 샘플 평가 데이터

`GoGoGolem/src/ai/evaluation/` 폴더에 실험 결과 데이터가 포함되어 있습니다.

| 파일 | 내용 |

| `hint_disguise_results.json` | 5개 테스트 케이스에 대한 LLM 생성 편지 원문, 점수(1~5), 위장 평가, 누설 평가, 채점 근거 |
| `hint_disguise_scores.png` | 케이스별 힌트 위장도 점수 bar chart (전체 평균선 포함) |
