# Ground Rules

## 1. 커뮤니케이션 규칙

### 1.1. 사용하는 채널

- **카카오톡**
  - 일정, 숙제, 간단한 피드백 전달에 사용
  - 파일 전송은 되도록 스크린샷 형태
  - 주요 알림 및 공지 전달

- **Notion**
  - 회의록 작성 및 관리
  - 작업 공간 관리

- **Jira**
  - 개발 기능 명세화
  - 기능별, 업무별 진행상황 공유

### 1.2. 회의 관련 규칙

**정기 회의 일정**
  - 격주 금요일 오후 2시 ~ 5시
  - 필요 시 토요일 오후 1시 ~ 4시 대면/비대면 진행

**회의 진행 규칙**
  - 지각은 10분까지 허용 (온라인 회의 포함)  
    - 지각 3회 시 커피 구매
  - 불참 시 사전 카카오톡 보내기

- **회의실(공대 109호) 은별이 예약**

---

## 2. 작업 방식

### 2.1. 회의 진행 방식
- 다음 회차 아젠다 결정 및 작성
- 논의할 때는 논의사항을 회의록에 소제목으로 작성하고 그 아래에 동시 기록하며 회의
- 회의 중 떠오르는 테스크는 바로 기록
- 회의 중 떠오르는 타 아젠다는 바로 기록해두고, 현 논의에서 벗어나지 않도록 함
- 회의 종료 후 공지사항을 카카오톡방에 공유

### 2.2. 핵심 기획의 의사 결정 방식

- 각 팀원이 즐거움을 느끼는 방향성의 키워드를 최대한 포괄하는 게임을 기획
- 만장일치 원칙 적용: 팀원 전원의 긍정(설득 포기 포함)이 있을 때만 결정
- 의견 차이: 의견이 다를 경우, 먼저 팀원들이 모두 동의하는 기준·핵심가치를 되짚고 이것을 기준으로 각 의견을 평가함. 이를 통해 합의 가능한 지점을 모색함.
- 표현 및 용어 통일: 결정된 사안에 대해서는 팀 내에서 사용하는 표현과 어휘를 통일하거나 각자 표현이 다르더라도 의미한 바가 같음을 재차 확인한다.

## 3. 역할 분담 및 책임

**역할과 전문성 존중의 원칙: 각 담당자가 해당 영역에 대해 모든 결정 권한이 있다.** 단, 결정 사항에 대해 타 분야 담당자와 기술적 충돌이 있는 경우 1:1 면담을 진행하여 해결안 또는 합의점을 모색한다.

| 강한나(리더) | 박세은 | 최은별 |
| --- | --- | --- |
| PM, 퍼실리테이터 |  | 기획자 |
| Unity 엔지니어, 데이터 엔지니어 | AI 엔지니어, 인프라 아키텍트 | Unity 엔지니어 |
| 서기 | 보조 서기 |  |
| 회의 알리미 |  | 수업 알리미 |
- **리더**: 대외적 업무
- **PM**: 팀 내 리소스 관리와 회의 주도
- **기획자**: 게임 기획을 포함한 게임 개발 전반의 전문가
- **엔지니어**: 개발 업무
- **인프라 아키텍트**: 서버(AWS) 비용 관리 및 sa-aa 담당
- **서기**: Notion, Jira 관리
- **보조 서기**: NotionAI, Jira 셋업 등 보조 관리
- **회의 알리미**: 회의 장소, 숙제 등 주요 공지사항을 카카오톡방에 전달
- **수업 알리미**: 수업 내용 요약, 제출해야 하는 과제 정리

## 4. 팀 내 약속

- 부정적인 화법(패닉, 사과, 좌절 등) 자제
- 팀원 상호 응원 및 격려
- 모르는 사항은 항상 질문
  - 이해가 되지 않을 경우 설명자의 책임으로 간주, 자유롭게 질문할 수 있는 분위기 조성
- 협업 요청 건을 최우선순위로 처리
  - 협업 요청 시 구체적인 테스크와 기한을 명시
